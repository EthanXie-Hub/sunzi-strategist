# 我把《孙子兵法》做成了一个先算胜、再出手的战略 Skill

今天开源一个我持续打磨的 AI Skill：**孙子·战略诊断器**。

![孙子·战略诊断器](../assets/promo/01-cover.png)

它不是一个会引用《孙子兵法》的聊天提示词。

我真正想解决的是：当一个人面对竞争、谈判、市场进入、组织僵局或高代价选择时，AI 能不能先帮他把账算清，再决定是否出手。

## 为什么很多“战略建议”没有用

很多战略建议听起来正确：

- 做差异化。
- 建立壁垒。
- 聚焦优势。
- 顺势而为。
- 不要打价格战。

问题是，它们往往没有回答：

- 你不靠运气，凭什么能赢？
- 哪个未知信息最可能推翻结论？
- 什么才算赢，哪些代价不能付？
- 应该在哪打，明确不打哪里？
- 对手回应后，你下一步怎么接？
- 到什么信号必须撤？

如果没有这些，战略就只是更高级的鼓励。

## 把《孙子兵法》当作决策算账框架

![五层战略诊断](../assets/promo/02-framework.png)

这个 Skill 用五层完成一次战略诊断：

1. **先胜 · 五事七计**：不靠运气，凭什么能赢？
2. **全胜 · 伐谋**：什么才算赢？怎样避免惨胜？
3. **形势 · 虚实**：先守住什么，再只打哪个薄弱切口？
4. **九变 · 因敌**：对手如何反制，局势变化后如何动？
5. **五危 · 六败**：情绪、面子、轻敌和战线过宽会不会毁掉计划？

每次输出必须给明确判断、决定性未知、主战场、不打清单、行动打法、对手反制、止损线和翻盘条件。

## 最重要的，不是教人硬刚

《孙子兵法》的最高目标不是战斗本身，而是以更小代价拿到更完整的结果。

所以这个 Skill 会主动提醒：

- 不要在对手最强处攻城。
- 不要为了面子烧掉现金流、声誉和退路。
- 不要把所有资源铺满所有战线。
- 不要把欺骗信任关系、报复和操纵包装成战略。
- 不要在关键事实未知时用假数据算出虚假的“胜”。

真正成熟的战略，常常是缩小战场、先侦察、借力、谈判、等待窗口，或者干脆不打。

![真实战略问题](../assets/promo/03-real-case.png)

## 它为什么必须有止损线

很多计划失败，不是因为最初判断完全错误，而是错误信号出现后仍继续投入。

因此，Skill 强制要求每个建议写明：

- 哪个指标证明打法有效。
- 哪个对手反制出现时要换打法。
- 到什么时间、投入或损失必须停止。
- 哪个事实会推翻当前结论。

这让战略不再是一次性的漂亮分析，而是一套可验证、可撤回的行动判断。

## 如何使用

安装后可以直接这样问：

```text
$sunzi-strategist
我们资源少于成熟竞品，但想进入他们占优的市场。
请给我决定性未知、主战场、对手反制和止损线。
```
